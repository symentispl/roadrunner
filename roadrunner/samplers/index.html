Samplers

Roadrunner is sampler-agnostic. Each sampler is a pluggable implementation that executes one request per virtual-thread iteration and returns a ProtocolResponse with nanosecond-precision timestamps.

Built-in samplers

The following samplers are bundled with the standard distribution:

Sampler Description

vm

In-VM baseline — sleeps for a fixed number of milliseconds. Use this to measure Roadrunner’s own overhead.

ab

HTTP benchmarking, API-compatible with Apache’s ab tool.

zero

Zero-latency no-op. Useful for measuring the absolute ceiling throughput of the engine.

Plugin samplers

The following samplers are distributed as separate plugin JARs. Install them by placing the JAR under ~/.roadrunner/plugins/protocols/.

Sampler Description

jdbc

Executes a SQL query against any JDBC-compatible database.

neo4j

Executes a Cypher query against a Neo4j database.

Writing your own sampler

Implement io.roadrunner.api.samplers.SamplerProvider from the roadrunner-samplers-spi module, annotate it with @Command (picocli) for CLI options, and register it via META-INF/services/io.roadrunner.api.samplers.SamplerProvider or a module-info.java provides directive.

See VmSamplerProvider and AbSamplerProvider in the source for reference implementations.
